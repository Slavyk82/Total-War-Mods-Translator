# Configurable Pack Prefix Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a Settings/General option that lets the user override the hardcoded `!!!!!!!!!!` pack prefix used in generated `.pack` filenames and the `.loc` files inside packs.

**Architecture:** Store the prefix as a string setting (default `!!!!!!!!!!`). The export/compilation orchestration reads it from `SettingsService` and threads it as an explicit parameter into the naming functions, each keeping the literal default for backward compatibility. Detection/parsing of existing files stays on the standard prefix (generation-only change). A new General-tab section edits the value, with a load-order warning, a sanitizer, a live preview, and a reset button.

**Tech Stack:** Flutter, Riverpod (codegen), slang (i18n), mocktail (tests), Result type.

**Spec:** `docs/superpowers/specs/2026-06-09-configurable-pack-prefix-design.md`

/// Removes characters that are illegal in Windows filenames from a pack prefix.
///
/// The pack prefix is concatenated into `.pack` and `.loc` filenames, so any
/// of `/ \ : * ? " < > |` would produce invalid paths. Everything else
/// (including an empty string) is preserved per design: no strong validation.
String sanitizePackPrefix(String input) {
  return input.replaceAll(RegExp(r'[/\\:*?"<>|]'), '');
}
```

  /// Generate output .loc path with language prefix (all lowercase)
  ///
  /// Transforms the original .loc path to include the configured load-order
  /// [prefix]. Example with the default prefix:
  /// text/db/Something.loc -> text/db/!!!!!!!!!!_fr_twmt_something.loc
  static String buildLocInternalPath(
    String sourceLocFile,
    String langLower, {
    String prefix = AppConstants.defaultPackPrefix,
  }) {
    // Get the directory and filename parts
    final dir = path.dirname(sourceLocFile).replaceAll('\\', '/').toLowerCase();
    final fileName = path.basename(sourceLocFile);

    // Remove .loc extension if present and convert to lowercase
    String baseName = fileName;
    if (baseName.toLowerCase().endsWith('.loc')) {
      baseName = baseName.substring(0, baseName.length - 4);
    }
    baseName = baseName.toLowerCase();

    // Build new filename with prefix (all lowercase)
    final newFileName = '${prefix}_${langLower}_twmt_$baseName.loc';

    // Combine directory and new filename
    if (dir.isEmpty || dir == '.') {
      return newFileName;
    }
    return '$dir/$newFileName';
  }
```

    String baseName,
    String langCode, {
    String prefix = AppConstants.defaultPackPrefix,
  }) {
    final upperLang = langCode.toUpperCase();

    // Remove an existing prefix if present. Recognize BOTH the standard
    // prefix and the configured one so a file generated with a custom prefix
    // is not double-prefixed.
    String cleanName = baseName;
    for (final existing in {'!!!!!!!!!!', prefix}) {
      if (existing.isNotEmpty && baseName.startsWith('${existing}_')) {
        final parts = baseName.split('_');
        if (parts.length >= 3) {
          cleanName = parts.sublist(2).join('_');
        }
        break;
      }
    }

    // Ensure .loc extension
    if (!cleanName.toLowerCase().endsWith('.loc')) {
      cleanName = '$cleanName.loc';
    }

    return '${prefix}_${upperLang}_$cleanName';
  }
